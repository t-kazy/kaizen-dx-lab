# 社長の頭の中を効率的に引き出す — ナレッジ抽出戦略

## なぜ「一つ一つ洗い出し」は得策でないか

```
❌ 従来型アプローチ：
社長に「知っていること全部書き出してください」
→ 社長の時間を大量に奪う
→ 何を書けばいいかわからない
→ 途中で止まる
→ 結局やらなくなる

✅ 今回のアプローチ：
社長の日常業務の中から自然に引き出す
＋ AIを使って短時間で大量に引き出す
```

---

## 戦略：3つの引き出し方を組み合わせる

```
方法①               方法②                方法③
AIインタビュー    ×  現場で質問が来たら   ×  既存資料を
（集中抽出）         記録する（日常抽出）     AIに読ませる

最初の1〜2回で       継続的に              すでにある情報を
大枠を引き出す       細かい知識が貯まる     活用する
```

---

## 方法①：AIインタビュー（最も効率的・最初にやるべき）

### 概要
社長に1〜2時間話してもらうだけで、大量のナレッジを構造化できる。
あなたが聞き役になり、録音 → AI文字起こし → AI整理。

### 具体的な進め方

#### セッション1回目（60〜90分）：業務の全体像

```
【聞く順番】

① まず大きな流れを聞く
「案件が入ってから完了するまで、ざっくりどんな流れですか？」

  受注 → 現場調査 → 見積 → 材料手配 → 施工 → 検査 → 請求

② 各フェーズを深掘り
「現場調査では具体的に何を見ますか？」
「見積はどうやって作ってますか？」
「材料の発注先はどう決めてますか？」

③ トラブル・例外を聞く
「今までで一番困った現場はどんなケースでしたか？」
「新人がよくやるミスってありますか？」
「これだけは絶対守らないとダメ、ってルールはありますか？」
```

#### セッション2回目（60分）：よく聞かれること

```
【質問リスト】

「社員からよく聞かれる質問TOP10は？」
「新人が最初に覚えるべきことは？」
「技能実習生に最初に教えることは何になりそう？」
「安全面で特に気をつけていることは？」
「うちのやり方の特徴・こだわりは？」
```

### ツールとワークフロー

```
┌──────────┐    ┌───────────┐    ┌───────────┐    ┌───────────┐
│ 社長に    │    │ スマホで   │    │ AIで      │    │ AIで      │
│ 話しても  │───→│ 録音      │───→│ 文字起こし │───→│ 構造化    │
│ らう      │    │           │    │           │    │           │
│ 60〜90分  │    │ ボイス    │    │ Gemini    │    │ ChatGPT/  │
│           │    │ メモ等    │    │           │    │ Claude    │
└──────────┘    └───────────┘    └───────────┘    └───────────┘
```

#### Step 1：録音
- スマホのボイスメモで録音（特別な機材不要）
- Google Pixel なら「レコーダー」アプリが文字起こし付き

#### Step 2：文字起こし
- **Gemini**（Google AI Studio / 無料枠あり）に音声ファイルをアップロード
  - 「この音声を文字起こししてください」
- または **Google ドキュメント**の音声入力機能
- 精度が高い順：Gemini > 専用ツール > Google音声入力

#### Step 3：AIで構造化
録音テキストをChatGPTまたはClaudeに渡して整理してもらう。

```
【プロンプト例】

以下はダクト工事会社の社長へのインタビュー書き起こしです。
この内容から以下の形式でナレッジを抽出・整理してください。

1. FAQ形式（Q&A）に整理
2. カテゴリ分け（作業手順/安全/会社ルール/工具・材料/その他）
3. 新人向け・技能実習生向けに特に重要な項目にマークを付ける
4. 不明瞭な箇所は「要確認」と記載

---
（書き起こしテキストを貼り付け）
```

### 期待できるアウトプット

```
1回のインタビュー（90分）から得られるもの：

・FAQ 30〜50件
・業務フロー図 1本
・新人向け「最初に覚えること」リスト
・安全ルール一覧
・よくあるトラブルと対処法 10〜20件

→ これだけでDifyチャットボットの初期ナレッジとして十分
```

## 方法②：日常業務からの自動蓄積（継続的）

方法①で大枠を取った後、日常の中で細かいナレッジを拾い続ける。

### 仕組み：社長への質問を自動記録

```
【現状】
社員「社長、〇〇ってどうすればいいですか？」
社長「△△だよ」
→ 消える

【改善後】
社員「社長、〇〇ってどうすればいいですか？」
社長「△△だよ」
社員 → Google Formに記録（30秒）
→ Spreadsheetに蓄積
→ 月1回あなたがFAQに整理
```

### もっと楽にする方法：Google Chat + あなたの後処理

```
社長がGoogle Chatで指示・回答する
→ やりとりが自動で残る
→ あなたが月1回チャット履歴を見てFAQに転記
  （AIに「このチャット履歴からFAQを抽出して」と頼める）
```

**ポイント：社長の行動を変えない。周りの仕組みで拾う。**

## 方法③：既存資料をAIに読ませる

### ありそうな既存資料

社長は「何もない」と思っていても、実は散らばっている資料がある。

```
探すべきもの：
□ 過去の見積書（Excel / 紙）    → 単価・工数のナレッジ
□ 安全書類（過去提出分）         → 安全ルールのナレッジ
□ 施工写真（スマホ / PC）        → 作業手順のナレッジ
□ メーカーのカタログ             → 材料知識
□ 元請けからの施工要領書         → 作業基準
□ 社長のスマホのLINE履歴        → 指示・回答のナレッジの宝庫
□ 過去の日報（紙でもあれば）      → 作業記録
```

### やること
1. 初回セッションで「何かしら紙やデータで残っているものはありますか？」と聞く
2. 見つかったものをスキャン or 写真撮影
3. NotebookLM or ChatGPTに投入
4. 「この資料から社内FAQを作って」と指示

## 推奨する進め方：まとめ

### 初回セッション後〜1ヶ月目

```
Week 1:
├── Google Workspace契約・初期設定
└── 既存資料の収集を社長にお願い
    「あるものだけでOK、探さなくていい」

Week 2:
├── 社長インタビュー 第1回（90分）
│   テーマ：業務全体の流れ・よくある質問
├── 録音 → 文字起こし → AI構造化
└── FAQ 30〜50件 + 業務フロー完成

Week 3:
├── 社長インタビュー 第2回（60分）
│   テーマ：トラブル事例・安全・技能実習生向け
├── 録音 → 文字起こし → AI構造化
└── 追加FAQ + 安全ルール + 新人向けリスト完成

Week 4:
├── NotebookLMに全資料を投入 → 社長に試してもらう
├── Google Chat開始 → 以後の指示・QAが自動蓄積
└── Difyチャットボットの初期版構築開始
```

### 社長の拘束時間

```
従来型：「全部書き出して」      → 数十時間（現実的に無理）
今回：  インタビュー2回 + α    → 合計3〜4時間で80%カバー

内訳：
・インタビュー 第1回    90分
・インタビュー 第2回    60分
・既存資料を探す        30分
・NotebookLMを試す     30分
──────────────────────────
合計                   約3.5時間
```

「ナレッジの整理はこちらでやります。
 社長にお願いしたいのは2つだけです。

 ① 2回、1時間ずつ話を聞かせてください（録音します）
 ② 手元にある書類やデータを見せてください

 それだけで社内QAチャットボットの8割は作れます。
 あとは日々の業務の中で自然に増えていく仕組みを作ります。」
