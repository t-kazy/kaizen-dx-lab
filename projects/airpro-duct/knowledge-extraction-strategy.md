# 社長の頭の中を効率的に引き出す — ナレッジ抽出戦略

## なぜ「一つ一つ洗い出し」は得策でないか

```
❌ 従来型アプローチ：
社長に「知っていること全部書き出してください」
→ 社長の時間を大量に奪う
→ 何を書けばいいかわからない
→ 途中で止まる
→ 結局やらなくなる

✅ 今回のアプローチ：
社長の日常業務の中から自然に引き出す
＋ AIを使って短時間で大量に引き出す
```

---

## 戦略：3つの引き出し方を組み合わせる

```
方法①               方法②                方法③
AIインタビュー    ×  現場で質問が来たら   ×  既存資料を
（集中抽出）         記録する（日常抽出）     AIに読ませる

最初の1〜2回で       継続的に              すでにある情報を
大枠を引き出す       細かい知識が貯まる     活用する
```

---

## 方法①：AIインタビュー（最も効率的・最初にやるべき）

### 概要
社長に1〜2時間話してもらうだけで、大量のナレッジを構造化できる。
あなたが聞き役になり、録音 → AI文字起こし → AI整理。

### 具体的な進め方

#### セッション1回目（60〜90分）：業務の全体像

```
【聞く順番】

① まず大きな流れを聞く
「案件が入ってから完了するまで、ざっくりどんな流れですか？」

  受注 → 現場調査 → 見積 → 材料手配 → 施工 → 検査 → 請求

② 各フェーズを深掘り
「現場調査では具体的に何を見ますか？」
「見積はどうやって作ってますか？」
「材料の発注先はどう決めてますか？」

③ トラブル・例外を聞く
「今までで一番困った現場はどんなケースでしたか？」
「新人がよくやるミスってありますか？」
「これだけは絶対守らないとダメ、ってルールはありますか？」
```

#### セッション2回目（60分）：よく聞かれること

```
【質問リスト】

「社員からよく聞かれる質問TOP10は？」
「新人が最初に覚えるべきことは？」
「技能実習生に最初に教えることは何になりそう？」
「安全面で特に気をつけていることは？」
「うちのやり方の特徴・こだわりは？」
```

### ツールとワークフロー（Plaud活用）

```
┌──────────┐    ┌──────────────┐    ┌───────────┐    ┌───────────┐
│ 社長に    │    │ Plaud で     │    │ Plaudが    │    │ AIで      │
│ 話しても  │───→│ 録音        │───→│ 自動で     │───→│ 深く構造化 │
│ らう      │    │              │    │ 文字起こし │    │           │
│ 60〜90分  │    │ ポケットに   │    │ ＋要約     │    │ ChatGPT/  │
│           │    │ 入れるだけ   │    │ ＋マインド  │    │ Claude    │
│           │    │              │    │   マップ   │    │           │
└──────────┘    └──────────────┘    └───────────┘    └───────────┘
```

#### Step 1：Plaudで録音
- Plaudをポケットや胸ポケットに入れてインタビュー開始
- 普段から使い慣れているのでオペレーション負荷ゼロ
- 社長との普段の打ち合わせや現場での会話もついでに録っておける

#### Step 2：Plaudの自動処理（文字起こし＋要約）
録音後、Plaudアプリが自動で以下を生成：

```
【Plaudが自動で出すもの】
├── 文字起こし（話者ラベル付き）  ← 誰が何を言ったか区別できる
├── AI要約                       ← 要点が自動整理される
└── マインドマップ               ← 話題の構造が可視化される
```

#### Step 3：Plaudからエクスポート

```
【エクスポート形式】
├── 文字起こし → TXT / SRT / DOCX / PDF（タイムスタンプ・話者ラベル付き可）
├── 要約      → TXT / Markdown / DOCX / PDF
├── マインドマップ → JPEG / Markdown
└── 音声      → MP3 / WAV
```

**おすすめのエクスポート方法：**
- 文字起こし → **TXT**（AIに渡しやすい）
- 要約 → **Markdown**（そのままナレッジとして使える）
- 音声 → **MP3**（バックアップ用にGoogle Driveへ保存）

#### Step 4：AIで深く構造化
Plaudの文字起こし＋要約をChatGPTまたはClaudeに渡して、さらに整理。

```
【プロンプト例】

以下はダクト工事会社の社長へのインタビューの文字起こしと要約です。
この内容から以下の形式でナレッジを抽出・整理してください。

1. FAQ形式（Q&A）に整理
2. カテゴリ分け（作業手順/安全/会社ルール/工具・材料/その他）
3. 新人向け・技能実習生向けに特に重要な項目にマークを付ける
4. 不明瞭な箇所は「要確認」と記載

---
■ 文字起こし：
（PlaudからエクスポートしたTXTを貼り付け）

■ 要約：
（Plaudの要約を貼り付け）
```

**ポイント：文字起こしと要約の両方を渡すことで精度が上がる。**
要約で全体像を掴み、文字起こしで詳細を拾える。

### 期待できるアウトプット

```
1回のインタビュー（90分）から得られるもの：

・FAQ 30〜50件
・業務フロー図 1本
・新人向け「最初に覚えること」リスト
・安全ルール一覧
・よくあるトラブルと対処法 10〜20件

→ これだけでDifyチャットボットの初期ナレッジとして十分
```

## 方法②：日常業務からの自動蓄積（継続的）

方法①で大枠を取った後、日常の中で細かいナレッジを拾い続ける。

### 最強パターン：Plaudで日常会話をそのまま蓄積

社長は普段から社員に口頭で指示・教育している。その会話自体がナレッジ。

```
【現状】
社員「社長、〇〇ってどうすればいいですか？」
社長「△△だよ」
→ 消える

【Plaud活用】
社員「社長、〇〇ってどうすればいいですか？」
社長「△△だよ」（Plaudが録音中）
→ Plaudが自動で文字起こし＋要約
→ あなたが月1〜2回エクスポートしてFAQに追加
```

**社長は普段通り話すだけ。Plaudがポケットに入っているだけで知識が蓄積される。**

### Plaud日常録音の運用ルール（提案）

```
・現場での指導場面 → 録音ON
・社員からの質問対応 → 録音ON
・朝礼・打ち合わせ → 録音ON
・プライベートな雑談 → 録音OFF（社長の判断）

※ 社員には「業務改善のために記録させてもらいます」と事前に伝える
```

### 月次ナレッジ更新フロー

```
┌─────────────────────────────────────────────────────┐
│ 月1〜2回のあなたの作業（1〜2時間）                      │
│                                                      │
│ 1. Plaudアプリで今月の録音一覧を確認                    │
│ 2. 業務に関係ある録音をエクスポート                      │
│    ・文字起こし → TXT                                  │
│    ・要約 → Markdown                                   │
│ 3. AIに渡して新しいFAQを抽出                            │
│ 4. 既存FAQシートに追加                                  │
│ 5. Difyのナレッジベースを更新                           │
└─────────────────────────────────────────────────────┘
```

### 補助手段：Google Chat + Google Forms

Plaudで拾えない場面（テキストでのやりとり等）は従来の方法で補完。

```
・Google Chatでの指示・回答 → やりとりが検索可能な状態で残る
・Google Forms「教えてメモ」 → 社員が聞いたことを30秒で記録
```

**ポイント：Plaudがメイン、Chat/Formsがサブ。社長の行動は一切変えない。**

## 方法③：既存資料をAIに読ませる

### ありそうな既存資料

社長は「何もない」と思っていても、実は散らばっている資料がある。

```
探すべきもの：
□ 過去の見積書（Excel / 紙）    → 単価・工数のナレッジ
□ 安全書類（過去提出分）         → 安全ルールのナレッジ
□ 施工写真（スマホ / PC）        → 作業手順のナレッジ
□ メーカーのカタログ             → 材料知識
□ 元請けからの施工要領書         → 作業基準
□ 社長のスマホのLINE履歴        → 指示・回答のナレッジの宝庫
□ 過去の日報（紙でもあれば）      → 作業記録
```

### やること
1. 初回セッションで「何かしら紙やデータで残っているものはありますか？」と聞く
2. 見つかったものをスキャン or 写真撮影
3. NotebookLM or ChatGPTに投入
4. 「この資料から社内FAQを作って」と指示

## 推奨する進め方：まとめ

### 初回セッション後〜1ヶ月目

```
Week 1:
├── Google Workspace契約・初期設定
└── 既存資料の収集を社長にお願い
    「あるものだけでOK、探さなくていい」

Week 2:
├── 社長インタビュー 第1回（90分）
│   テーマ：業務全体の流れ・よくある質問
├── 録音 → 文字起こし → AI構造化
└── FAQ 30〜50件 + 業務フロー完成

Week 3:
├── 社長インタビュー 第2回（60分）
│   テーマ：トラブル事例・安全・技能実習生向け
├── 録音 → 文字起こし → AI構造化
└── 追加FAQ + 安全ルール + 新人向けリスト完成

Week 4:
├── NotebookLMに全資料を投入 → 社長に試してもらう
├── Google Chat開始 → 以後の指示・QAが自動蓄積
└── Difyチャットボットの初期版構築開始
```

### 社長の拘束時間

```
従来型：「全部書き出して」      → 数十時間（現実的に無理）
今回：  インタビュー2回 + α    → 合計3〜4時間で80%カバー

内訳：
・インタビュー 第1回    90分
・インタビュー 第2回    60分
・既存資料を探す        30分
・NotebookLMを試す     30分
──────────────────────────
合計                   約3.5時間
```
