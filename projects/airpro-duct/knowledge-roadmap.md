# ナレッジゼロからのナレッジ蓄積ロードマップ — (株)AIR PRO ダクト

## 前提

ダクト工事会社にとって最大のリスクは**ベテラン（＝社長）の退職・不在＝知識の消滅**。
技能実習生が入ればさらに「教える負荷」が増える。

### ナレッジの特徴
- **社長が全てのナレッジを持っている**（一極集中型）
- 一つ一つ洗い出して文書化する方法は社長の時間を奪い、現実的でない
- → **AIインタビュー方式で短時間に一気に引き出す**のが最適解
- 詳細は [knowledge-extraction-strategy.md](./knowledge-extraction-strategy.md) を参照

---

## 全体像：4ステップで段階的に積み上げる

```
Step 0            Step 1          Step 2           Step 3            Step 4
社長の頭を      → 置き場を作る  → 日常業務から    → ナレッジを      → AIが答える
AIで引き出す                      自然に貯まる       整理・検索可能に

録音+AI文字起こし  Google Drive    Google Forms     NotebookLM        Dify
ChatGPT/Claude                    Google Sheets                      チャットボット
Gemini(文字起こし)                 Google Chat

[Week 1-3]        [Week 1-2]     [Week 3〜]       [Month 2〜]       [Month 2-3〜]
```

---

## Step 0：社長の頭の中をAIで引き出す（Week 1〜3）★最重要

### 考え方
「全部書き出してください」は無理。代わりに**話してもらうだけ**でいい。
録音 → AI文字起こし → AI構造化で、社長の拘束時間は合計3〜4時間。

### やること
1. 社長インタビュー2回（90分 + 60分）を録音
2. Geminiで文字起こし
3. ChatGPT / Claudeで FAQ・業務フロー・安全ルールに構造化

### 期待アウトプット
- FAQ 30〜50件
- 業務フロー図
- 安全ルール一覧
- よくあるトラブルと対処法

→ **これだけでDifyチャットボットの初期ナレッジとして十分**

詳細な進め方は [knowledge-extraction-strategy.md](./knowledge-extraction-strategy.md) を参照。

- **社長や事務の方と一緒にフォルダを作る**（自分たちの場所という意識）

### あなたの作業

- 共有ドライブ作成＋フォルダ構成
- 全員のアカウント設定・スマホへのDriveアプリ導入

## Step 2：日常業務からナレッジが自然に貯まる仕組み（2〜4週間）

### 考え方
「ナレッジを書いてください」は続かない。
**普段の業務をするだけで情報が残る仕組み**を作る。

### 施策 2-A：Google Formsで日報

```
【日報フォーム項目（例）】
- 日付（自動）
- 現場名（プルダウン）
- 作業内容（選択式＋自由記述）
- 使用材料・数量
- 困ったこと・気づいたこと  ← ★ここがナレッジになる
- 現場写真（添付）
- 明日の予定
```

- Forms回答 → **自動でSpreadsheetに蓄積**
- 「困ったこと・気づいたこと」欄が自然とナレッジになっていく
- 現場写真も日付・現場名と紐づいて保存される

### 施策 2-B：LINE → Google Chat移行

- 業務連絡をGoogle Chatのスペースに集約
- スペースの例：「全体連絡」「〇〇現場」「事務連絡」
- **やりとりが検索可能な状態で残る**
- LINEと使い勝手は近いので移行ハードルは低い

### 施策 2-C：「教えてメモ」の習慣化

ベテランに聞いたことを簡単に記録する仕組み。

```
【教えてメモ Google Form】
- 質問した人
- 誰に聞いた？
- 何を聞いた？（質問）
- 答え（教えてもらった内容）
- カテゴリ（作業手順/工具/安全/その他）
```

- 「聞いたらメモする」だけ
- 回答がSpreadsheetに溜まる → そのままFAQの素材になる

### あなたの作業
- Google Forms作成（日報・教えてメモ）
- Google Chatスペース設定
- 使い方の簡単な説明会（30分程度）

## Step 3：貯まったナレッジを整理・検索可能にする（1〜2ヶ月）

### やること
Step 2で自然に貯まったデータを「使える形」に整理する。

### 施策 3-A：FAQ一覧の整理

日報の「困ったこと」と「教えてメモ」から、繰り返し出てくるQ&Aを抽出。

```
【FAQ Spreadsheet構成】
| カテゴリ | 質問 | 回答 | 補足・写真リンク | 対象者 |
|---------|------|------|----------------|--------|
| 作業手順 | 角ダクトの吊り間隔は？ | 1.5m以内... | Drive写真リンク | 全員 |
| 安全 | 高所作業の安全帯ルールは？ | ... | 安全ルール.pdf | 全員 |
| 会社 | 有給の申請方法は？ | ... | | 全員 |
| 工具 | ニブラの刃の交換頻度は？ | ... | 写真リンク | 新人 |
```

### 施策 3-B：NotebookLM で社内ドキュメント検索

- **Google NotebookLM**（無料）にDriveの資料をアップロード
- 社内資料に対して自然言語で質問できる
- 「ダクトの保温材の種類は？」→ 該当資料から回答
- **Difyより先にこれを試す価値がある**（無料・Google連携・導入が簡単）

### 施策 3-C：作業手順書の作成

よく聞かれる作業をベテランに実演してもらい、動画 or 写真＋テキストで手順書化。

```
【手順書の作り方】
1. ベテランに作業してもらう
2. スマホで動画撮影（5分以内）
3. Googleドキュメントに写真+手順を記載
4. Drive「02_作業手順」フォルダに保存
5. FAQシートにリンクを追加
```

### あなたの作業
- FAQ Spreadsheet作成（日報データから抽出）
- NotebookLM設定・資料アップロード
- 手順書テンプレート作成
- ベテランへのインタビュー補助（録音→テキスト化はAIで）

## Step 4：AIチャットボットで誰でもアクセス可能に（2〜3ヶ月）

### やること
Step 1〜3で蓄積・整理されたナレッジをDifyチャットボットに接続。

### 構成イメージ

```
┌─────────────┐    ┌─────────────┐    ┌──────────────────┐
│  社員/実習生  │───→│ Dify チャット │───→│ ナレッジソース     │
│  （スマホ）   │    │  ボット      │    │ ・FAQ Sheets      │
│              │←───│              │←───│ ・作業手順書       │
│  質問を入力   │    │  回答を生成   │    │ ・安全ルール       │
│  （多言語OK） │    │  （多言語対応）│    │ ・会社ルール       │
└─────────────┘    └─────────────┘    └──────────────────┘
```

### Difyの設定

```
【ナレッジベース（Knowledge）に入れるもの】
1. FAQ一覧（CSV or テキスト）       ← Step 3で整理済み
2. 作業手順書（PDF / テキスト）      ← Step 3で作成済み
3. 安全ルール（PDF）                ← Step 1でDriveに保存済み
4. 会社ルール・就業規則（PDF）       ← Step 1でDriveに保存済み
```

### 多言語対応の実装

```
【Difyチャットフロー設計】

[ユーザー入力]
     ↓
[言語判定] → 日本語 / ベトナム語 / 中国語 / etc.
     ↓
[ナレッジ検索] → FAQや手順書から該当情報を取得
     ↓
[回答生成] → 検出した言語で回答を返す
     ↓
[ユーザーに表示]
```

- ベトナム語で「ダクトの取り付け方は？」と聞けばベトナム語で回答
- 日本語で聞けば日本語で回答
- プロンプトで言語対応を指示するだけで実現可能

### あなたの作業
- Difyアプリ構築（チャットフロー）
- ナレッジベースへのデータ投入
- 多言語プロンプト設計
- テスト＋フィードバック収集

## 成功のための3原則

### 1. 「書け」ではなく「入力するだけ」

```
❌ 「ナレッジを文書化してください」    → 誰もやらない
✅ 「日報の最後に1行、気づいたこと書いて」→ 自然に貯まる
```

### 2. 最初は量より継続

```
❌ 完璧なマニュアルを一気に作る         → 挫折する
✅ 毎週1個FAQが増えれば、1年で50個      → 十分なナレッジベース
```

### 3. 「使われている実感」を見せる

```
❌ 貯めるだけで誰も見ない              → やる意味を感じなくなる
✅ 「先月の日報からこんなことがわかりました」→ 貯める動機になる
   「チャットボットがこう答えてくれます」   → 自分の知識が役立つ実感
```

## タイムライン全体像

```
Month 1     Month 2     Month 3     Month 4     Month 5     Month 6
|-----------|-----------|-----------|-----------|-----------|-----------|
[Step1: 置き場]
            [Step2: 自然に貯まる仕組み        ]
                        [Step3: 整理・検索               ]
                                    [Step4: AIチャットボット        ]
                                                          [運用・改善→]
```

各Stepは完全に順番通りでなく、並行して進めてOK。
**Step 0（社長インタビュー）とStep 1（置き場）を最初の2〜3週で同時に進める**のがベスト。
Step 0で引き出したナレッジをStep 1の置き場に入れることで、すぐにStep 3〜4に進める。
