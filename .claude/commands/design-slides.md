# /design-slides — プレゼンテーションスライド生成

ユーザーの指定に基づき、ハイクオリティなHTMLスライドデッキを生成するスキル。

## パラメータ（ユーザー入力から抽出）

- **トピック**: プレゼンのメインテーマ
- **対象**: 想定する聴衆（経営層、技術者、顧客など）
- **トーン**: プレゼンの雰囲気（データドリブン、カジュアル、堅実、ビジョナリーなど）
- **スライド数**: 生成するスライドの枚数（デフォルト: 12）
- **テーマ**: ビジュアルテーマ名（下記参照）

## 利用可能テーマ

### arctic-frost（寒色系・クリーン）
- カラー: #0c2d4a → #2e86c1 → #17a2b8
- 特徴: フロスト調グラデーション、透明感のあるカード、氷のような清潔感
- 用途: 経営提案、データ分析報告、技術戦略

### ember-glow（暖色系・エネルギッシュ）
- カラー: #7b241c → #e74c3c → #f39c12
- 特徴: 炎のグラデーション、力強いアクセント、高コントラスト
- 用途: 営業プレゼン、新製品発表、モチベーション系

### forest-calm（グリーン系・ナチュラル）
- カラー: #0e6251 → #1abc9c → #82e0aa
- 特徴: 自然な落ち着き、アースカラー、サステナビリティ感
- 用途: ESG報告、ヘルスケア、環境関連

### midnight-pro（ダーク系・プロフェッショナル）
- カラー: #1a1a2e → #16213e → #0f3460
- 特徴: ダークモード、ネオン風アクセント、モダン
- 用途: テック系プレゼン、スタートアップ、デモ

## スライド構成ルール

### 必須スライド
1. **カバー** — タイトル、サブタイトル、宛先、日付
2. **エグゼクティブサマリー** — 3つのKPIメトリクスと要約文
3. **セクション区切り** — 各パートの冒頭にナンバリング付きの中扉
4. **まとめ・ネクストステップ** — 最終スライドにアクションアイテム

### トーン別の構成ガイド

**データドリブン:**
- 必ず定量データ（%、金額、時間）をメトリクスで大きく表示
- CSS棒グラフ、比較テーブル、ROIフローチャートを活用
- インサイトボックスでデータの意味を解説
- Before/After比較を含める

**カジュアル:**
- アイコン・絵文字を多用
- カード型レイアウト中心
- 質問形式のヘッダー

**堅実:**
- テーブルとリスト中心
- 控えめな装飾、データ重視
- リスクと対策を必ず含める

**ビジョナリー:**
- 大きなメトリクスとインパクトのあるステートメント
- タイムライン/ロードマップを視覚的に
- 未来志向の言葉遣い

## デザイン品質基準

### レイアウト
- CSS Grid/Flexboxで構造化（.two-col, .three-col）
- スライドごとに適切なレイアウトを選択（1カラム/2カラム/3カラム/カード）
- 余白を十分に確保（padding: 56px 72px）
- 1スライドの情報量を制限（箇条書き最大5項目）

### タイポグラフィ
- Google Fonts: Noto Sans JP（300/400/500/700/900）
- h1: 3.2em（カバーのみ）
- h2: 2em（スライドタイトル）
- 本文: 1.15em、行間2.0

### データビジュアライゼーション（CSS-only）
- **メトリクス**: .metric-value（グラデーションテキスト、3.5em）
- **棒グラフ**: .bar-chart > .bar-row（ラベル + トラック + バリュー）
- **比較テーブル**: 角丸テーブル、ゼブラストライプ
- **ROIフロー**: .roi-flow（ボックス → 矢印 → 結果ボックス）
- **フェーズバー**: .phase-bar > .phase-segment
- **ドーナツチャート**: CSS conic-gradient
- **リスクマトリクス**: 色分けタグ付きリスト

### コンポーネント
- **カード**: .card + バリアント（--frost, --teal, --warm, --alert）
- **インサイトボックス**: .insight（左ボーダー付きハイライト）
- **タグ**: .tag（ラベル表示用ピル型）
- **ステップ**: .step-num + .step-row（番号付き手順）
- **トップバー**: .slide-topbar（グラデーション装飾ライン）

### インタラクション
- キーボード: ←→ / Space / Enter / Backspace
- タッチ: スワイプ対応
- フルスクリーン: Fキー
- スライドカウンター表示

### 対象者別の調整

**経営層向け:**
- ROI・コスト効果を前面に
- エグゼクティブサマリーを充実
- リスクと対策を必ず含める
- ネクストステップに具体的な承認アクション

**技術者向け:**
- アーキテクチャ図・構成図
- 技術選定の根拠
- PoC結果の詳細データ

**顧客向け:**
- 課題→解決→効果のストーリー
- 事例・実績ベース
- わかりやすい言葉遣い

## 出力

- 単一のHTMLファイルとして出力
- CSSは`<style>`タグ内にインライン
- JavaScriptはナビゲーション用のみ最小限
- フォントはGoogle Fonts CDNから読み込み
- 外部ライブラリ不使用（Pure CSS/JS）
