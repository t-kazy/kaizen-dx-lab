# Dify チャットフロー構成ガイド（カテゴリ選択型ナレッジ検索）

ユーザーが13カテゴリから1つ選択 → 質問入力 → 該当カテゴリのナレッジのみ検索して回答するチャットフロー。

---

## アーキテクチャ概要

```
[開始（カテゴリ選択 + 質問入力）]
  ↓
[条件分岐（カテゴリ判定）]
  ├─ カテゴリ01 → [ナレッジ検索01] ─┐
  ├─ カテゴリ02 → [ナレッジ検索02] ─┤
  ├─ ...                            ├→ [LLM回答生成] → [回答]
  └─ カテゴリ13 → [ナレッジ検索13] ─┘
```

### この構成のメリット

- **検索ノイズの排除**: 選択カテゴリのナレッジだけを検索するため、無関係なチャンクが混入しない
- **検索精度の向上**: 1つのナレッジ内での検索なので、Top Kの枠を有効に使える
- **ユーザー体験**: カテゴリを選ぶことで、何について聞けるか明確になる

---

## ノード別設定

### 1. 開始ノード

2つの入力変数を定義する。

| 変数名 | 型 | 設定 |
|--------|------|------|
| `category` | **セレクト（単一選択）** | 13カテゴリの選択肢 |
| `sys.query` | テキスト（デフォルト） | ユーザーの質問文 |

#### カテゴリ選択肢の設定

開始ノードの変数設定で、`category` をセレクト型にし、以下の選択肢を定義する：

| 値（value） | 表示名（label） |
|-------------|-----------------|
| `01` | 会社概要・理念・沿革 |
| `02` | 組織・部署 |
| `03` | 就業規則・勤怠 |
| `04` | 休暇制度 |
| `05` | 経費精算・出張 |
| `06` | IT機器・アカウント |
| `07` | セキュリティポリシー |
| `08` | コミュニケーションツール |
| `09` | 入社時手続き |
| `10` | 福利厚生 |
| `11` | 評価制度・目標設定 |
| `12` | コンプライアンス |
| `13` | よくある質問（FAQ） |

> **Dify設定手順**: 開始ノード → 「+」で変数追加 → 型を「セレクト」 → 変数名 `category` → 選択肢を上記の通り追加

---

### 2. 条件分岐ノード（IF/ELSE）

`category` の値に応じて、対応するナレッジ検索ノードにルーティングする。

#### Difyでの実装方法（2つの方式）

**方式A: IF/ELSEノードを連結する（推奨）**

```
[IF category == "01"] → [ナレッジ検索01]
  ↓ ELSE
[IF category == "02"] → [ナレッジ検索02]
  ↓ ELSE
[IF category == "03"] → [ナレッジ検索03]
  ↓ ELSE
  ...（繰り返し）
```

- 各IFノードの条件: `{{#start.category#}}` が `01` と等しい
- シンプルで分かりやすい
- ノード数は多くなるが、Difyの標準機能だけで実装可能

**方式B: コードノードでルーティング（上級者向け）**

Pythonコードノードを使って、カテゴリ値をそのまま出力変数にし、後続のナレッジ検索で動的に使う。ただしDifyの現仕様ではナレッジ検索のdataset_idを動的に切り替えるのは難しいため、**方式Aを推奨**。

---

### 3. ナレッジ検索ノード（×13個）

カテゴリごとに1つずつ、計13個のナレッジ検索ノードを作成する。

#### 各ノードの設定

| 設定項目 | 値 |
|---------|-----|
| ナレッジ | そのカテゴリに対応する**1つのナレッジ**のみ |
| クエリ | `{{#start.sys.query#}}`（ユーザーの質問文） |

#### 検索パラメータ（全ノード共通）

| 設定項目 | 推奨値 | 説明 |
|---------|--------|------|
| 検索モード | **ハイブリッド検索** | ベクトル+全文検索 |
| Top K | **5** | 1ナレッジなので5で十分 |
| Score Threshold | **0.3** | 低めに設定して取りこぼし防止 |
| 重み配分 | セマンティック **0.7** / キーワード **0.3** | |

> **ポイント**: 全ナレッジ横断検索と違い、1ナレッジだけを検索するのでTop Kは少なめでOK。
> その分、該当チャンクの精度が高くなる。

#### ナレッジ設定（アップロード時）

| 設定項目 | 推奨値 | 説明 |
|---------|--------|------|
| インデックスモード | **高品質** | エコノミーは精度が低い |
| Embeddingモデル | **text-embedding-3-small** 以上 | |
| チャンク設定 | **カスタム** | 自動分割は使わない |
| セパレータ | `##` | H2見出し単位で分割 |
| 最大チャンクサイズ | **1000** | トークン数 |
| チャンクオーバーラップ | **100** | 前後の文脈を保持 |

---

### 4. LLMノード（回答生成）

13個のナレッジ検索ノードの出力を**すべてこの1つのLLMノードに接続**する。
Difyでは複数のノードからの接続が可能で、実際に実行されたノードの出力だけが渡される。

#### モデル設定

| 設定項目 | 推奨値 |
|---------|--------|
| モデル | gpt-4o / claude-sonnet-4-20250514 等 |
| Temperature | 0.3 |
| Max Tokens | 2048 |

#### コンテキスト接続

13個のナレッジ検索ノードそれぞれの `result` を、LLMノードのコンテキスト入力に接続する。

#### システムプロンプト

```
あなたは社内ナレッジに基づいて質問に回答するアシスタントです。
ユーザーは「{{#start.category#}}」カテゴリについて質問しています。

## ルール
1. 提供されたコンテキスト情報のみに基づいて回答してください
2. コンテキストに該当する情報がない場合は「選択されたカテゴリ内に該当する情報が見つかりませんでした。別のカテゴリを選択するか、質問を変えてお試しください。」と伝えてください
3. 回答の根拠となるセクション名を【参照元】として明示してください
4. 推測や外部知識による補完は行わないでください
5. 箇条書きや表を使って分かりやすく回答してください

## コンテキスト情報
{{#context#}}
```

---

### 5. 回答ノード

LLMノードの出力をそのまま返す。

```
{{#llm.text#}}
```

## トラブルシューティング

### 「該当コンテキストがない」と表示される場合

**Step 1: 正しいナレッジ検索ノードが実行されているか確認**
- デバッグモードで実行し、条件分岐が正しく動いているか確認
- 選択したカテゴリに対応するナレッジ検索ノードが実行されているか

**Step 2: ナレッジ検索ノードの出力を確認**
- 実行されたナレッジ検索ノードの出力にチャンクが返ってきているか
- チャンクが0件 → ナレッジ設定の問題（Step 3へ）
- チャンクがある → LLMノードの接続の問題（Step 4へ）

**Step 3: ナレッジ設定を確認**
1. ナレッジのドキュメントが「有効」になっているか
2. インデックスの構築が完了しているか
3. チャンクが正しく分割されているか（ナレッジ > ドキュメント > チャンク一覧で確認）
4. Score Thresholdを **0.2** まで下げてテスト

**Step 4: LLMノードの接続を確認**
1. 該当のナレッジ検索ノードの出力がLLMのコンテキストに接続されているか
2. `{{#context#}}` がシステムプロンプトに含まれているか

### IF/ELSE分岐が正しく動かない場合

- `category` 変数の型が「セレクト」になっているか確認
- 条件の比較値がvalue（`01`等）と一致しているか確認（label名ではなくvalue値で比較）

## Dify上での構築手順（ステップバイステップ）

### Step 1: ナレッジを13個作成

1. Difyの「ナレッジ」画面で「ナレッジを作成」
2. 各カテゴリのMDファイルをアップロード
3. インデックスモード: 高品質
4. Embeddingモデル: text-embedding-3-small
5. チャンク設定: カスタム → セパレータ `##` → 最大1000トークン → オーバーラップ100
6. 13個すべて作成完了するまで繰り返す

### Step 2: チャットフローを作成

1. 「スタジオ」→「アプリを作成」→「チャットフロー」
2. 開始ノードに `category`（セレクト型）変数を追加し、13個の選択肢を設定

### Step 3: 条件分岐を構築

1. 開始ノードの後に「IF/ELSE」ノードを追加
2. 条件: `{{#start.category#}}` == `01`
3. TRUE → ナレッジ検索01ノードを接続
4. ELSE → 次のIF/ELSEノードへ
5. 13カテゴリ分繰り返す

### Step 4: ナレッジ検索ノードを13個作成

1. 各IF分岐のTRUE側に「ナレッジ検索」ノードを追加
2. 対応するナレッジ1つだけを選択
3. 検索パラメータを設定（ハイブリッド検索、Top K: 5、Score Threshold: 0.3）

### Step 5: LLMノードと回答ノードを接続

1. 13個のナレッジ検索ノードすべてからLLMノードに接続
2. LLMノードのコンテキスト入力に各ナレッジ検索ノードのresultを設定
3. システムプロンプトを設定
4. LLMノードから回答ノードに接続

### Step 6: テスト

1. デバッグモードで各カテゴリを選択してテスト
2. ナレッジ検索ノードの出力にチャンクが返ってきているか確認
3. 回答内容が適切か確認
