# ナレッジMDファイル フォーマットガイド

Difyのナレッジ検索精度を最大化するためのMDファイル作成ルール。

## 基本原則

Difyはナレッジファイルを「チャンク」に分割してベクトル化する。
チャンクの質 = 検索精度 なので、MDの書き方がそのまま検索精度に直結する。

---

## 1. 見出し構造ルール

### セパレータとして `##` を使う

Difyのチャンク分割は `##`（H2見出し）単位で行うのが最も安定する。

```markdown
# ドキュメントタイトル（ファイルに1つだけ）

## トピックA
このセクションの内容...

## トピックB
このセクションの内容...
```

**NG例：** 見出しなしで長文を書くと、1チャンクが巨大になり検索精度が下がる。

### 1チャンク = 1トピック

1つの `##` セクションには1つのトピックだけを書く。
複数のトピックが混在すると、検索時にノイズになる。

---

## 2. チャンクサイズの目安

| 項目 | 推奨値 |
|------|--------|
| 1セクションの文字数 | 200〜800文字 |
| 最大文字数 | 1000文字以内 |
| 最小文字数 | 100文字以上 |

短すぎると文脈が失われ、長すぎるとノイズが増える。

---

## 3. メタデータの埋め込み

各セクションの冒頭に、検索キーワードとなるメタ情報を入れる。

```markdown
## 有給休暇の申請方法

**カテゴリ**: 人事・労務
**関連キーワード**: 有給, 休暇, 申請, 休み, PTO

有給休暇を申請するには、以下の手順で...
```

これにより、ユーザーがどんな言い回しで質問しても、キーワードマッチでヒットしやすくなる。

---

## 4. Q&A形式の活用

よくある質問は、Q&A形式で書くと検索精度が上がる。
ユーザーの質問文とナレッジの質問文が近くなるため。

```markdown
## Q: 経費精算の締め日はいつですか？

**カテゴリ**: 経理
**関連キーワード**: 経費, 精算, 締め日, 締切

A: 経費精算の締め日は毎月25日です。
25日を過ぎた場合は翌月の精算となります。
```

---

## 5. 表・リストの使い方

表やリストは検索にヒットしやすいが、単独だと文脈が失われる。
必ず説明文と組み合わせる。

```markdown
## 各部署の連絡先一覧

社内の各部署への連絡先は以下の通りです。

| 部署名 | 内線番号 | メールアドレス |
|--------|---------|---------------|
| 総務部 | 1001 | soumu@example.com |
| 人事部 | 1002 | jinji@example.com |
```

---

## 6. ファイル分割の指針

13個のナレッジファイルを効果的に分割するための指針。

| ファイル | 内容例 |
|---------|--------|
| 01-company-overview.md | 会社概要・理念・沿革 |
| 02-organization.md | 組織図・部署説明 |
| 03-hr-rules.md | 就業規則・勤怠ルール |
| 04-leave-policy.md | 休暇制度（有給・特別休暇等） |
| 05-expense.md | 経費精算・出張ルール |
| 06-it-guide.md | IT機器・アカウント管理 |
| 07-security-policy.md | セキュリティポリシー |
| 08-communication.md | コミュニケーションツール利用ルール |
| 09-onboarding.md | 入社時手続き |
| 10-benefits.md | 福利厚生 |
| 11-evaluation.md | 評価制度・目標設定 |
| 12-compliance.md | コンプライアンス・ハラスメント対策 |
| 13-faq.md | よくある質問（横断的） |

### ポイント

- ファイル名にプレフィックス番号をつけて管理しやすくする
- 1ファイルのサイズが大きくなりすぎたら分割する（目安：50セクション以下）
- 横断的なFAQファイルを1つ用意すると、検索のカバレッジが上がる

---

## 7. 避けるべきパターン

| NG | 理由 | 改善方法 |
|----|------|---------|
| 見出しなしの長文 | チャンクが巨大になる | `##` で分割 |
| 画像のみのセクション | テキスト検索できない | alt textや説明文を追加 |
| 略語のみの記述 | 検索にヒットしない | 正式名称も併記 |
| 同じ内容の重複 | ノイズになる | 1箇所にまとめて相互参照 |
| 空のセクション | 無駄なチャンクが生成される | 削除する |
