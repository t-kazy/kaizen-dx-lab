# シフトデータ → freee勤怠インポート 作業手順書

## 概要

カイトKANNABEの出勤予定表（CSV）を、freee勤怠管理Plusにインポートできる形式に自動変換するツール。

---

## 毎月の作業手順

### ステップ1: シフト表CSVを準備する

出勤予定表のCSVファイルを `sample_data/` フォルダに配置する。

ファイル名の例: `202605_カイトKANNABE_出勤予定表.csv`

### ステップ2: 変換コマンドを実行する

#### コマンドプロンプトを開く

1. キーボードの `Windowsキー + R` を押す
2. `cmd` と入力して `Enter` を押す
3. 黒い画面（コマンドプロンプト）が開く

#### プロジェクトフォルダに移動する

```
cd %USERPROFILE%\kaizen-dx-lab
```

※ プロジェクトの場所が違う場合は、実際のフォルダパスに置き換えること。

#### 変換コマンドを実行する

```
python -m src.excel_to_csv "sample_data/202605_カイトKANNABE_出勤予定表.csv" --format freee
```

→ 同じフォルダに `202605_カイトKANNABE_出勤予定表_freee.csv` が生成される。

#### うまく動かない場合

| 症状 | 対処 |
|------|------|
| `python` が認識されない | `py` に置き換えて試す（例: `py -m src.excel_to_csv ...`） |
| モジュールが見つからない | `pip install -r requirements.txt` を実行してから再度試す |

### ステップ3: freeeにインポートする

1. freee勤怠管理Plus にログイン
2. 勤怠データインポート画面を開く
3. 生成された `_freee.csv` ファイルをアップロード

---

## スタッフの変更があった場合

`config/freee_mapping.json` の `employee_map` を編集する。

```json
"employee_map": {
    "名前": "freeeの従業員コード"
}
```

- **スタッフ追加**: 新しい行を追加
- **スタッフ削除**: 該当行を削除
- 従業員コードはfreeeの従業員一覧で確認

## 休日の自動分類

シフト表の「公休」「休日」「休み」は、日付に応じて自動的に分類される：

| 条件 | 分類 |
|------|------|
| 祝日（元日、建国記念日など）に該当する休み | **法定外休日** |
| 日曜日の休み | **法定休日** |
| 月〜土の休み（祝日以外） | **法定休日** |

### 設定方法

`config/freee_mapping.json` の `holiday_patterns` にfreeeのパターンコードを設定する：

```json
"holiday_patterns": {
    "法定休日": "ここにfreeeのパターンコードを入力",
    "法定外休日": "ここにfreeeのパターンコードを入力"
}
```

※ パターンコードが未設定の場合、該当する休日はスキップされ警告が表示される。

### 休日扱いするシフト名

`holiday_shifts` で変更可能（デフォルト: 公休, 休日, 休み）。

## スキップされるシフト

以下のシフト名は変換時にスキップされる（freeeに出力しない）：

- 有休
- 欠勤

変更する場合は `config/freee_mapping.json` の `skip_shifts` を編集。

## トラブルシューティング

| 症状 | 原因・対処 |
|------|-----------|
| 「勤務日の日付が正しくありません」 | BOM付きファイルの可能性。最新版のツールを使用しているか確認 |
| 従業員コードが空欄 | `employee_map` に該当スタッフが未登録。追加する |
| パターンコードが空欄 | `time_to_pattern` に該当時間帯が未登録。追加する |
| 文字化け | 出力ファイルがcp932(Shift_JIS)で正常。Excelで開くと正しく表示される |
| 変換データが0件 | シフト表のフォーマットが変わった可能性。ヘッダー行を確認 |
